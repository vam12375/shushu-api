# 阿里云 + 宝塔面板部署指南（自建 GHCR 镜像版）

本文档指导你把**自己 fork 的 shushu-api 代码**部署到阿里云服务器。

**核心思路**：吃内存的镜像构建交给 GitHub Actions 免费完成，阿里云服务器（2C2G）只负责拉取镜像和运行容器，不参与编译，避免内存不足。

```
本地写代码 ──push──► GitHub ──Actions 构建──► GHCR 镜像仓库 ──pull──► 阿里云服务器运行
```

- **代码仓库**：`https://github.com/vam12375/shushu-api`
- **镜像地址**：`ghcr.io/vam12375/shushu-api:latest`

## 一、阿里云服务器准备

### 1. 服务器配置
- **配置**：2核2G（当前配置，够跑服务，不用于编译）
- **系统镜像**：推荐 **Ubuntu 22.04** 或 **Debian 12**
- **系统盘**：40G 以上

### 2. 安全组放行端口（关键）
阿里云控制台 → ECS → **安全组** → 入方向规则，放行：

| 端口 | 用途 |
|------|------|
| 22 | SSH 远程登录 |
| 8888 | 宝塔面板 |
| 80 / 443 | 网站 / HTTPS |
| 3000 | New-API 服务 |

> ⚠️ 阿里云有**两层防火墙**：安全组 + 宝塔自带防火墙。两边都要放行 3000，否则访问不了。

## 二、安装宝塔面板

SSH 登录服务器（Ubuntu/Debian）：

```bash
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && bash install.sh ed8484bec
```

安装完记下面板地址、用户名、密码，浏览器打开 `http://服务器IP:8888` 登录。要求宝塔 ≥ 9.2.0。

## 三、安装 Docker

1. 宝塔左侧菜单点 **Docker**
2. 首次进入提示安装 Docker，点 **立即安装**，等待完成

## 四、GitHub 构建镜像（首次）

> 这一步在你**本地电脑 + GitHub 网页**完成，服务器不参与。

### 1. 推送部署文件到 GitHub
在本地项目目录执行：

```powershell
git add .github/workflows/my-deploy.yml docker-compose.deploy.yml deploy/
git commit -m "新增 GHCR 自建部署配置"
git push origin main
```

### 2. 触发构建
1. 打开 `https://github.com/vam12375/shushu-api`
2. 顶部点 **Actions**
3. 左侧选 **My Deploy (GHCR amd64)**
4. 右侧 **Run workflow** → 选 `main` → 点绿色 **Run workflow**
5. 等待约 **8~15 分钟** 构建完成

### 3. 把镜像设为公开（只需一次）
构建成功后：
1. GitHub 个人主页 → **Packages** → 点进 `shushu-api`
2. **Package settings** → 底部 **Change visibility** → 设为 **Public**

> 设为 Public 后，服务器拉取镜像无需登录。

## 五、服务器部署 New-API

### 1. 创建目录
SSH / 宝塔终端：

```bash
mkdir -p /www/wwwroot/new-api && cd /www/wwwroot/new-api
```

### 2. 生成两个密钥
执行两次，分别记下结果：

```bash
openssl rand -hex 16   # 用作 SESSION_SECRET
openssl rand -hex 16   # 用作 CRYPTO_SECRET
```

### 3. 创建 docker-compose.yml
宝塔 **文件管理器**进入 `/www/wwwroot/new-api/`，新建 `docker-compose.yml`，粘贴以下内容，并修改所有标注「改成」的地方：

```yaml
version: '3.4'

services:
  new-api:
    image: ghcr.io/vam12375/shushu-api:latest
    container_name: new-api
    restart: always
    command: --log-dir /app/logs
    ports:
      - "3000:3000"
    volumes:
      - ./data:/data
      - ./logs:/app/logs
    environment:
      - SQL_DSN=root:改成你的MySQL密码@tcp(mysql:3306)/new-api
      - REDIS_CONN_STRING=redis://:改成你的Redis密码@redis:6379
      - SESSION_SECRET=改成第一个密钥
      - CRYPTO_SECRET=改成第二个密钥
      - TZ=Asia/Shanghai
      - ERROR_LOG_ENABLED=true
      - BATCH_UPDATE_ENABLED=true
      - NODE_NAME=new-api-node-1
    depends_on:
      - redis
      - mysql
    networks:
      - new-api-network
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O - http://localhost:3000/api/status | grep -o '\"success\":\\s*true' || exit 1"]
      interval: 30s
      timeout: 10s
      retries: 3

  redis:
    image: redis:latest
    container_name: redis
    restart: always
    command: ["redis-server", "--requirepass", "改成你的Redis密码"]
    networks:
      - new-api-network

  mysql:
    image: mysql:8.2
    container_name: mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: 改成你的MySQL密码
      MYSQL_DATABASE: new-api
      MYSQL_CHARACTER_SET_SERVER: utf8mb4
      MYSQL_COLLATION_SERVER: utf8mb4_unicode_ci
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - new-api-network

volumes:
  mysql_data:

networks:
  new-api-network:
    driver: bridge
```

**必改清单**（改错会启动失败）：

| 位置 | 说明 |
|------|------|
| `SQL_DSN` 里的密码 与 `MYSQL_ROOT_PASSWORD` | 两处填**同一个** MySQL 密码 |
| `REDIS_CONN_STRING` 里的密码 与 `redis-server` 的 | 两处填**同一个** Redis 密码 |
| `SESSION_SECRET` | 第一个 `openssl rand -hex 16` 结果 |
| `CRYPTO_SECRET` | 第二个 `openssl rand -hex 16` 结果 |

### 4. 启动

```bash
cd /www/wwwroot/new-api
docker-compose up -d
```

首次会拉取镜像，稍等。完成后访问：

```
http://你的阿里云公网IP:3000
```

**默认管理员 `root` / `123456`，登录后立刻改密码！**

### 5. 放上更新脚本（方便以后更新）
把仓库 `deploy/update.sh` 上传到 `/www/wwwroot/new-api/`，或直接新建该文件粘贴内容。

## 六、配置域名 + HTTPS（可选，推荐）

1. 域名解析 A 记录指向服务器公网 IP
2. 宝塔 → **网站** → 添加站点（填域名，不建库/FTP）
3. 站点设置 → **反向代理** → 添加：
   - 目标 URL：`http://127.0.0.1:3000`
   - 发送域名：`$host`
4. 站点设置 → **SSL** → Let's Encrypt 申请免费证书 → 开启「强制 HTTPS」

之后用 `https://你的域名` 访问，安全组里可关闭 3000 端口。

## 七、日常更新

改完代码后，**三步走**：

### 第 1 步：本地推代码
```powershell
git add .
git commit -m "你的改动说明"
git push origin main
```

### 第 2 步：GitHub 重新构建
两种方式任选其一：
- **网页手动**：Actions → My Deploy → Run workflow
- **打 tag 自动触发**：
  ```powershell
  git tag deploy-v1.0.1
  git push origin deploy-v1.0.1
  ```

等构建完成（约 10 分钟）。

### 第 3 步：服务器拉取更新
SSH 登录服务器：

```bash
cd /www/wwwroot/new-api
bash update.sh
```

脚本会自动：拉新镜像 → 滚动重启 → 清理旧镜像，**数据库和数据不会丢失**。

## 八、常见问题

### Q1：访问不了 3000 端口？
- 阿里云**安全组**放行 3000
- 宝塔 **安全** 里也放行 3000
- 检查容器是否在跑：`docker-compose ps`

### Q2：登录后频繁掉线？
检查 `SESSION_SECRET` 是否设置且非空。

### Q3：容器启动报错怎么排查？
```bash
docker-compose logs -f new-api
```
常见原因：Redis 密码两处不一致、没设 `CRYPTO_SECRET`、MySQL 密码不一致。

### Q4：服务器拉镜像提示无权限 / unauthorized？
说明 GHCR 镜像还是 Private。回到 [第四步第 3 小节](#3-把镜像设为公开只需一次)把镜像设为 Public。
若坚持用 Private，需先在服务器登录：
```bash
echo "你的GitHub_Token" | docker login ghcr.io -u vam12375 --password-stdin
```

### Q5：数据如何备份？
- 数据库：`mysql_data` Docker 卷
- 业务数据/日志：`/www/wwwroot/new-api/data` 和 `logs`
建议用宝塔「计划任务」定期打包这几个目录。

### Q6：构建很慢 / 失败？
GitHub Actions 免费额度足够，首次无缓存约 10~15 分钟，之后有缓存更快。失败时点开 Actions 里红色的步骤看日志。

cd /www/wwwroot/new-api
