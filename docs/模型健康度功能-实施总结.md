# 模型健康度（Model Health）功能实施总结

> 实施日期：2026-06-07　方案：A+C 融合（可用性为主 + 附带成功率）

## 1. 功能概述

在公开顶部导航的 **排行榜（Rankings）** 与 **文档（Docs）** 之间新增「**模型健康度**」入口，点击进入公开页面 `/model-health`，实时展示每个公开模型的可用性与调用质量：

- **可用性（主）**：每个模型的在线渠道数 / 总渠道数、平均探活响应时间、最近探活时间、健康状态（🟢在线 / 🟡降级 / 🔴离线 / ⚪未检测）。
- **成功率（附带）**：按所选周期（今日 / 本周 / 本月 / 全部）从请求日志聚合的调用成功率、调用次数、错误次数、平均耗时。

UI 采用 **Linear 风 Bento 状态大屏**（参考 awwwards / Apple / Linear 当下趋势）：

- **Hero 横幅**：大字号实时健康结论（全部正常 / N 个降级 / N 个离线）+ 脉冲呼吸圆点 + 随严重度变化的辉光背景 + 周期分段切换。
- **Bento 概览网格**：大小不一的瓦片——纯 SVG 甜甜圈环形图（在线 / 降级 / 离线 / 未检测占比，中心显示在线百分比）、状态计数瓦片（带状态色辉光描边）、模型数 / 渠道数 / 平均响应时间指标瓦片。
- **模型健康卡片网格**：玻璃拟态卡片，含厂商图标、状态徽章、成功率环形进度、渠道数与时延；hover 辉光浮起；顶部状态筛选 chips（全部 / 在线 / 降级 / 离线 / 未检测）+ 搜索框；`motion` stagger 入场动画；移动端自动堆叠为单列。

页面默认**公开**（无需登录），并可在「系统设置 → 维护 → 头部导航」中开关、设置是否需要登录访问。

## 2. 数据来源（复用现有数据，无新增采集逻辑）

| 维度 | 来源表 | 说明 |
| --- | --- | --- |
| 总渠道数 | `abilities` | 按 `model` 聚合 `COUNT(DISTINCT channel_id)`（含禁用） |
| 在线渠道数 / 平均响应时间 / 最近探活 | `abilities` JOIN `channels` | 仅取 `channels.status = 启用`，聚合 `COUNT(DISTINCT)`、`AVG(response_time)`、`MAX(test_time)` |
| 成功率 / 调用量 / 平均耗时 | `logs` | 成功=`type=2`(消费日志)，失败=`type=5`(错误日志)，均含 `model_name` |
| 模型清单 / 厂商图标 | `model.GetPricing()` + `model.GetVendors()` | 仅展示面向用户的公开模型 |

> 探活数据由既有「自动测试渠道」机制产出（`controller/channel-test.go`）。若管理员未开启自动测试，则相关模型显示「未检测（unknown）」。

### 健康状态判定逻辑

```
在线渠道数 == 0                      => 🔴 offline（离线）
在线渠道数 > 0 且 从未探活(test_time=0) => ⚪ unknown（未检测）
0 < 在线渠道数 < 总渠道数             => 🟡 degraded（降级）
在线渠道数 == 总渠道数               => 🟢 online（在线）
```

## 3. 三库兼容设计（SQLite / MySQL / PostgreSQL）

- 可用性聚合**拆成两条标准 SQL**，避免使用 `CASE WHEN` / `FILTER` / 布尔列进 SELECT 等跨库差异写法。
- 全程使用 GORM 的 `Table().Select().Group().Scan()`，仅用 `COUNT/AVG/MAX/COUNT(DISTINCT)` 等标准聚合函数。
- 未直接拼接保留字列；`model_name`、`channel_id`、`type`、`created_at` 等均为普通列。

## 5. 接口说明

```
GET /api/model_health?period=today|week|month|all
```

响应示例：

```json
{

    "models": [
      {
        "model_name": "gpt-4o",
        "vendor": "OpenAI",
        "vendor_icon": "https://...",
        "status": "online",
```

## 7. 注意事项

1. 平均响应时间统计的是**启用渠道**的探活耗时；从未探活的模型状态为「未检测」。
2. 成功率依赖请求日志，冷门 / 无流量模型在所选周期内显示「—」。
3. 接口结果带 **5 分钟缓存**，与排行榜一致，降低数据库压力。
4. 厂商图标字段 `vendor_icon` 为 **@lobehub/icons 图标标识**（非图片 URL），前端必须经 `getLobeIcon()` 渲染（内置首字母兜底），不可直接用 `<img>`。
5. 环形图使用纯 SVG 自绘（非 VChart），避免图表库异步主题初始化的加载竞态，且明暗主题自适应、无新增依赖。
6. 未改动任何受保护的 new-api / QuantumNous 品牌标识。
