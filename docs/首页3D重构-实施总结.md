# 首页沉浸式 3D 重构 - 实施总结

> 日期：2026-06-10
> 范围：仅首页（`web/default/src/features/home/`），其他页面不变
> 依赖：零新增（复用项目已有 `three@0.184`）

## 一、目标

在保持"鼠鼠🐭公益站"诙谐风格与纯色配色（禁渐变）的前提下，把首页 3D 从"氛围背景"升级为"视觉主角"，给人惊艳的第一印象。

设计方向已先通过静态原型确认：`docs/prototype/home-3d-preview.html`（可直接双击打开预览）。

## 四、关键技术决策

1. **零新依赖**：沿用 vanilla Three.js，不引入 R3F/drei（YAGNI），bundle 不增长。
2. **保留 LCP 优化**：`index.tsx` 的懒加载 + `requestIdleCallback` 挂载策略不动，3D chunk 不阻塞首屏标题渲染。
3. **emoji 当贴图**：🐭/🧀 用 canvas 画成 `CanvasTexture`，零外部图片请求（国内访问友好）。
4. **拖拽不抢交互**：`pointerdown` 在 window 层监听，命中 `button/a/input` 等交互元素或滚出 hero 区即不启动拖拽。
5. **preserve-3d 陷阱**：`overflow: hidden` 会强制 `transform-style: flat`，故看板卡片的 overflow 收进高光球专用裁剪层，卡片本体保持 preserve-3d，translateZ 分层才生效。
6. **高频动效直写 style**：标题倾斜、Tilt、3D 帧循环都绕过 React state，避免重渲染。
7. **确定性随机**：场景布局用种子随机，刷新不闪变。
8. **无障碍降级**：`prefers-reduced-motion` 下 3D 只渲染静态首帧，所有 CSS 动效直接显示最终状态。
9. **纯色约束**：所有"光感"用纯色 + blur/additive blending 实现，无渐变色（遵循既定设计偏好）。

## 六、手动验收要点

1. Hero 区按住空白处拖动 → 奶酪星球旋转，松手有惯性。
2. 向下滚动 → 星球滑向左后方、缩小让位，看板翻入登场。
3. 鼠标在看板上移动 → 卡片 3D 倾斜，内部两块面板分层悬浮，高光球跟随。
4. 切换深色模式 → 3D 整体降不透明度（75%），暖色不刺眼。
5. 移动端（<768px）→ 星球居中缩小，无拖拽冲突，滚动流畅。
